## Herkesin Kaçırdığı Nokta #1: Template Message Zorunluluğu

WhatsApp Business API, **rastgele mesaj gönderemezsiniz**. Alıcıya ilk mesajınız bir **template message** olmak zorunda — bu da Meta tarafından onaylanmış bir şablon.

**Ama gerçekte ne var?**
- Template onayı 24-48 saat sürebilir
- Birçok kategori (spor, politika, sağlık) reddedilebilir
- "Satış" kelimesi geçen şablonlar %40 ihtimalle reddediliyor

**Çözüm:** Unofficial API'ler (wbm, whatsapp-bulk-messenger) bu kısıtlamayı atlar — WhatsApp Web arayüzünü kullanır. Ancak risk: **hesap banlanabilir**.

## Herkesin Kaçırdığı Nokta #2: Numara Formatı Farkı

Türkiye'de +90 5XX XXX XX XX formatı gerekiyor. Birçok açık kaynak araç sadece uluslararası format kabul ediyor.

**wbm ve whatsapp-bulk-messenger:** Numara validation için farklı regex kalıpları kullanıyor. Yanlış format = mesaj gitmiyor, hata vermiyor.

### Seçenek 1: whatsapp-bulk-messenger (Python)
```bash
# 1. Klonla
git clone https://github.com/anirudhbagri/whatsapp-bulk-messenger.git
cd whatsapp-bulk-messenger

# 2. Bağımlılıkları yükle
pip install -r requirements.txt

# 3. WhatsApp Web'e tara
# Program çalışınca QR kod görünecek — WhatsApp mobil uygulamasından tarat

# 4. Excel dosyasına numaraları kaydet (ülke kodu dahil)
# Örnek: +905551234567

# 5. Mesaj metnini gir ve çalıştır
python send_bulk.py
```

### Seçenek 2: wbm (JavaScript)
```bash
# 1. Klonla
git clone https://github.com/Briuor/wbm.git
cd wbm

# 2. Bağımlılıkları yükle
npm install

# 3. Çalıştır
node index.js
# Tarayıcı açılacak — WhatsApp Web'e giriş yap

# 4. Mesaj ve alıcı listesi ayarla
```

## Kaç Mesaj Limiti Var, Nasıl Aşılır?

| Yöntem | Limit | Risk |
|--------|-------|------|
| Resmi API (Cloud) | 1000/24 saat/business hesabı | Düşük risk |
| WhatsApp Web unofficial | 50-100/1 saat | Orta risk (ban) |
| wbm ile session复用 | 200-500/1 saat | Yüksek risk |

**Unofficial API'lerde risk azaltma:**
- 30 saniye aralıklarla mesaj gönder
- Aynı numaraya tekrar deneme (2. deneme 1 saat sonra)
- Hesabı "business" olarak kullan
