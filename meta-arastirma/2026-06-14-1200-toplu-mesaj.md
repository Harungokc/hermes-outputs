## 1. Özet

WhatsApp Business API ile toplu mesaj gönderimi için 2026'da yeni nesil MCP (Model Context Protocol) sunucuları ortaya çıktı. WhatsApp'ın 24 saatlik müşteri etkileşim penceresi ve template message zorunluluğu, toplu mesaj araçlarını iki kategoriye ayırıyor: (1) Official API kullananlar ve (2) Web scraping/inofficial API kullananlar. Official API yöntemleri güvenilir ama maliyetli; unofficial yöntemler ücretsiz ama riskli.

**Herkesin Kaçırdığı Nokta:** WhatsApp bulk message araçları "spam bot" kategorisinde değerlendiriliyor ama asıl değer, kişiselleştirilmiş toplu mesajda. "Merhaba [isim], siparişiniz [urun] hazır" gibi template'ler ile %10-15 ek dönüşüm sağlanıyor. Teker teker göndermek ile toplu kişiselleştirilmiş mesaj arasında ciddi ROI farkı var.

## 3. Kişiselleştirilmiş Toplu Mesaj — Asıl Değer

### Resmi Yol: WhatsApp Business API Template Message

WhatsApp'ın resmi yolu template message kullanmak. Her mesaj bir önceden onaylanmış template'e dayanıyor.

**Avantajları:**
- 24 saatlik pencerede serbest mesajlaşma
- Yüksek deliverability
- Otomatik yanıt, sıralı mesajlaşma

**Dezavantajları:**
- Her template için onay süreci (24-48 saat)
- Mesaj başına ücret (~$0.05-0.15)
- Sadece aktif müşterilere gönderim

**Örnek Template (E-ticaret):**
```
Merhaba {{1}}! 👋

Siparişiniz {{2}} numaralı {{3}} ürünü hazırlandı.
Tahmini teslimat: {{4}}

Siparişinizi takip etmek için: {{5}}
```

### Unofficial Yollar (Riskli Ama Ücretsiz)

**wbm (WhatsApp Business unofficial API):**
- Ücretsiz
- Kullanıcı kendi WhatsApp hesabını kullanıyor
- Yüksek ban riski
- Sadece küçük ölçekli deneme için

**Kural:** Unofficial araçlar production ortamında KULLANILMAZ. Sadece prototype/test için.

## 4. Adım Adım Yapım Rehberi — WhatsApp Business API ile Kişiselleştirilmiş Toplu Mesaj

### Adım 1: WhatsApp Business API Erişimi

1. [Meta for Developers](https://developers.facebook.com/) → "My Apps" → Yeni uygulama oluştur
2. "Business" tipi seç → WhatsApp Business API'yi ekle
3. Test telefon numarası al (ücretsiz, sınırlı)

### Adım 2: Template Message Oluştur

1. [WhatsApp Business Manager](https://business.whatsapp.com/) → Messages → Templates
2. Yeni template oluştur (variables: `{{1}}`, `{{2}}`, vb.)
3. Onay bekle (24-48 saat)

### Adım 3: n8n ile Otomasyon

```n8n workflow:
Trigger (Schedule: Her gün 09:00)
↓
Google Sheets (Müşteri listesi oku)
↓
Code Node (Template'i kişiselleştir)
↓
HTTP Request (WhatsApp API'ye mesaj gönder)
```

### Adım 4: Claude Code ile Entegrasyon

```python
# n8n-MCP-Server ile Claude Code kullanımı
# WhatsApp mesaj gönderme
POST https://graph.facebook.com/v18.0/{Phone Number ID}/messages
{
  "messaging_product": "whatsapp",
  "to": "{recipient}",
  "type": "template",
  "template": {
    "name": "order_ready",
    "language": { "code": "tr" },
    "components": [
      {
        "type": "body",
        "parameters": [
          { "type": "text", "text": "Ayşe" },
          { "type": "text", "text": "#12345" },
          { "type": "text", "text": "Lüks Çanta" }
        ]
      }
    ]
  }
}
```

## 5. Herkesin Kaçırdığı Nokta #1 — Toplu Mesaj Değil, Sıralı Mesajlaşma

**Herkes ne yapıyor?** Toplu mesaj = tek seferde 1000 kişiye aynı mesaj.

**Ama gerçekte ne var?** En yüksek dönüşüm oranı, sıralı (sequential) mesajlaşmadan geliyor:
- 1. mesaj: "Merhaba! Siparişiniz hazır" (sipariş sonrası 1 saat)
- 2. mesaj: "Kargonuz yola çıktı" (24 saat sonra)
- 3. mesaj: "Kargonuz ulaştı mı?" (72 saat sonra)
- 4. mesaj: "%10 indirim kodu" (terk edilmiş sepethatırlatması)

Her adım müşterinin davranışına bağlı — bu zincirleme otomasyon, tek tek gönderimden 3-5x daha etkili.

## 6. Herkesin Kaçırdığı Nokta #2 — MCP Server'lar ile Meta Ads + WhatsApp Entegrasyonu

**Herkes ne düşünüyor?** "Meta Ads ve WhatsApp ayrı platformlar, entegre etmek zor."

**Ama gerçekte ne var?** Yeni nesil MCP server'lar (meta-ads-mcp, google-meta-ads-ga4-mcp) ile WhatsApp ve Meta Ads tek bir AI ajanından yönetilebilir. Örnek:
- Meta Ads'de "terk edilmiş sepet" kampanyası oluştur
- Aynı ajan WhatsApp'dan hatırlatma mesajı gönder
- Sonuçları tek dashboard'da gör
