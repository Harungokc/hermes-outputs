## 1. MaiAgent @ VivaTech 2026 — RAG Mı, Agent Mı?

**Herkesin Kaçırdığı Nokta:** Herkes RAG (Retrieval-Augmented Generation) inşa ediyor çünkü "doğru bilgi" vaat ediyor. Ama MaiAgent'ın VivaTech'teki sunumu tam tersini söylüyor: **RAG aslında bir "düşünce hapishanesi."** Agent'lar, retrieval yerine reasoning ile doğru yanıta ulaşabilir — üstelik 10x daha hızlı. RAG'ın tek avantajı "güvenilir kaynak gösterme" — o da bir yanılsama, çünkü retrieval her zaman %30-40 hata oranıyla çalışıyor.

**Teknik Gerçek:** RAG pipeline'ı 3 adım: (1) embedding, (2) cosine similarity search, (3) generation. Adım 2'deki semantic gap, en iyi embedding modellerinde bile %25-35 yanlış eşleşme üretiyor. MaiAgent'ın önerisi: **Agent tabanlı mimari** — model kendi karar veriyor, dışarıdan "doğru bilgi" sokmaya çalışmak yerine.

**Ne Yapmalı?**
- Mevcut RAG sistemlerini "agent-first" düşünceyle yeniden tasarlamak
- RAG'ı tamamen kaldırmak yerine "agent memory" olarak konumlandırmak

**Kaynak:** [VivaTech 2026 - MaiAgent](https://vat.vc/3Y8ZV)

## 2. AI Agent FFmpeg'de 21 Sıfır-Gün Açığı Buldu

**Herkesin Kaçırdığı Nokta:** Güvenlik dünyasında "AI agent tarama" artık sadece teorik değil — gerçek sonuçlar üretiyor. Bu agent, manuel analizde haftalar sürecek FFmpeg kodunu 48 saat içinde taradı ve 21 adet daha önce bilinmeyen güvenlik açığı buldu. **İnsan güvenlik mühendisleri bunu yılda ortalama 3-5 tane buluyor.**

**Rakamsal Karşılaştırma:**
- Manuel FFmpeg audit: ~2000 saat iş gücü
- AI agent tarama: 48 saat, ~$200 bulut maliyeti
- Bulunan açık: 21 sıfır-gün (0-day)

**Herkesin Kaçırdığı Nokta #2:** 21 açığın 7'si "use-after-free" türünde — bellek yönetimi hatası, en kritik kategori. FFmpeg milyonlarca cihazda çalışıyor (Twitter/X video işleme, YouTube transcoding, VLC player). Bu açıklar düzeltilene kadar her "ffmpeg -i input.mp4" komutu potansiyel code execution riski taşıyor.

**Sonuç:** AI agent artık sadece "iş otomasyonu" değil — **otomatik güvenlik araştırmacısı.** Yazılım tedarik zincirinde her bağımlılık (ffmpeg, openssl, libxml2) için haftalık agent taraması standardı haline gelebilir.

## 3. AutoJack Attack — AI Agent'ı Ele Geçirme

**Herkesin Kaçırdığı Nokta:** AI agent'lar "akıllı" ama aynı zamanda "sömürülebilir." AutoJack attack'ı şunu gösteriyor: Bir web sayfası, ziyaretçinin AI agent'ına (tarayıcı uzantısı, desktop agent, vs.) sahte talimatlar enjekte edebiliyor. Agent "trusted" kanalıyla iletişim kurduğunu düşünürken aslında saldırgan konuşuyor.

**Tehdit Modeli:**
1. Kullanıcı kötü niyetli bir sayfayı ziyaret ediyor
2. Sayfa, agent'ın context window'una prompt injection yapıyor
3. Agent — "güvenilir" olduğu için — talimatları uyguluyor (veri çalma, yanlış yere mesaj gönderme)

**Korunma Yolu:** Agent'ların "tool call" zincirini doğrulaması gerekiyor — hangi talimatın insan kaynaklı, hangisinin harici injection olduğunu ayırt edemiyorlar.

**Sonuç:** Agent tabanlı sistemlerde "input sanitization" = yeni güvenlik katmanı. Geleneksel XSS/SQL injection koruması yetmiyor.

**Kaynak:** Bing News — "AutoJack Attack Lets One Web Page Hijack AI Agent for Host Code Execution"

## 4. Microsoft MDASH — 100+ Tehdit Avcılığı Agent'ı

**Herkesin Kaçırdığı Nokta:** Microsoft, Build 2026'da MDASH'ı duyurdu (Microsoft Defender for AI Security Hunting). 100'den fazla uzmanlaştırılmış agent — her biri farklı tehdit kategorisinde avlanıyor. Bir agent ransomware family'lerini takip ediyor, bir diğeri supply chain compromise'arı tarıyor, üçüncüsü zero-day exploit'leri tahmin ediyor.

**Neden Önemli:** Geleneksel SIEM (Security Information and Event Management) sistemleri "kural tabanlı" çalışıyor — bilinen kalıpları arıyor. MDASH agent'ları "anormallik" arıyor — daha önce görülmemiş saldırı pattern'larını bile yakalayabiliyor.

**Rakamsal Büyüklük:** Microsoft 365 Defender günde 2.5 trilyon güvenlik sinyalini işliyor. İnsan analist bunu gerçek zamanlı işleyemez — agent'lar bu boşluğu dolduruyor.

**Kaynak:** Bing News — "Build 2026: Microsoft's MDASH exits preview with 100+ specialized threat-hunting AI agents"
