Şirketler ajan tasarrufunda "ne kadar AI gerekiyor?" sorusunu soracak.

Stripe'ın payment take rate'ine benzer bir model geliyor.

**Herkesin Kaçırdığı Nokta #3 — E-ticaret Ajanları İçin Kritik:** Instagram → WhatsApp → sipariş sistemi çalıştıranlar için: ajan ne kadar "yardımcı" olursa (ürün öner, kargo takip et, şikayet çöz) = o kadar çok AI token harcar = maliyet artar. ## 3. Sophos: Claude Code, Cursor, Codex LOLBin Güvenlik Açığı

**Ne oldu:** Sophos araştırması = AI coding agent'lar (Claude Code, Cursor, Codex) Windows endpoint security kurallarını tetikliyor. LOLBin (Living Off the Land Binary) saldırı kalıplarına benzer davranışlar sergiliyorlar.

**Herkesin Kaçırdığı Nokta #1 — "Yasal Araç = Saldırı Aracı Gibi Görünüyor":** AI coding agent'lar legitimate iş yapıyor ama endpoint security bunları "saldırgan" olarak işaretliyor. Neden? Agent'lar dosya yazıyor, komut çalıştırıyor, credential access yapıyor — tam da bir saldırganın yapacağı şeyler. Security team'ler ajanları false positive olarak blokluyor = ajan çalışmıyor.

**Herkesin Kaçırdığı Nokta #2 — GhostApproval Açığı Genişliyor:** Daha önce işlediğimiz GhostApproval flaw (6 AI coding assistant'ı etkiliyordu) + yeni Sophos bulgusu = AI coding tool güvenliği = aktif exploit yüzeyi. Agent'lar code review, testing, deployment yaparken security kurallarını tetikliyor. Enterprise'lar ajan deploy edemiyor çünkü security team reddediyor.

**Herkesin Kaçırdığı Nokta #3 — E-ticaret Ajanları İçin Uyarı:** WhatsApp Business + n8n + Claude Code ile ajan geliştirenler = local IDE'de (VS Code alternatifi) çalışırken bu sorunla karşılaşabilirsiniz. Şirketin endpoint security'si ajanınızı "malware" olarak işaretleyebilir. "Ajan güvenliği" = artık sadece server tarafı değil — development environment tarafı da kritik.
