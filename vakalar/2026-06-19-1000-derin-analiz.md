## 1. cloakward/cloak — API Key'leri Agent'lardan Gizleme

### Ne Yapıyor?
Rust ile yazılmış local vault sistemi. AI agent'larınızın API key'lerini kullanmasına izin veriyor ama model hiçbir zaman gerçek key'i görmüyor. SLSA ve Cosign ile imzalı, MCP uyumlu.

### Herkesin Kaçırdığı Nokta
**"Model görmüyor" = sadece UI aldatmacası DEĞİL.** Gerçek güvenlik: Model API'ye istek atarken, vault proxy olarak çalışıyor ve raw key'i asla prompt context'ine koymuyor. Bu, "environment variable gizleme"den farklı — environment variable'lar hâlâ process memory'de görünür. Cloak'un yaklaşımı: model sadece "bu API'ye erişimim var" sinyalini alıyor, nasıl eriştiğini bilmiyor.

### Herkesin Kaçırdığı Nokta #2
**MCP uyumluluğu = yeni standart.** Model Context Protocol (MCP) artık agent framework'ler arası standard haline geliyor. Cloak'un MCP uyumu = Claude Code, Codex, Cursor, LangChain, CrewAI — hepsinde çalışabilir. Vendor lock-in yok.

### Teknik Detaylar
- **Dil:** Rust
- **Topics:** cli, security, vault, mcp, secrets, ai-agents, local-first, slsa, cosign
- **Oluşturulma:** 2026-05-05
- **Güncelleme:** Aktif
- **License:** Açık kaynak

### Gerçek Kullanım Senaryoları
- Enterprise: Geliştirici makinelerinde API key'lerin agent'larla paylaşılması
- Güvenlik gereksinimi olan projeler: finans, sağlık, GDPR
- CI/CD pipeline'larında agent'ların sadece belirli API'lere erişmesi

**Kaynak:** https://github.com/cloakward/cloak

## 2. Persona.js — WebMCP-Native Agent UI Library

### Ne Yapıyor?
Vanilla JavaScript ile agent arayüzü oluşturmaya yarayan açık kaynak kütüphane. WebMCP'yi native olarak destekliyor — herhangi bir backend gerektirmeden agent'ın web sayfanızı sürmesini sağlıyor.

### Herkesin Kaçırdığı Nokta
**"Vanilla JS" = her yerde çalışır.** React, Vue, Svelte gibi framework bağımlılığı yok. WebMCP ile agent'ınız sayfadaki form'ları doldurabilir, arama yapabilir, rezervasyon oluşturabilir — sanki insan kullanıcıymış gibi. Bu, e-ticaret sitelerinde Instagram→WhatsApp→Agent akışı için kritik — kullanıcı DM'den agent'a ulaşıyor, agent web sitesinde işlem yapıyor.

### Herkesin Kaçırdığı Nokta #2
**MIT lisans = ticari kullanım serbest.** Birçok agent UI kütüphanesi ya GPL ya da proprietary. Persona.js MIT — şirketinizde自由自在 kullanabilirsiniz.

### Teknik Detaylar
- **Dil:** JavaScript (vanilla)
- **Özellik:** WebMCP-native, SSE backend desteği
- **Demo'lar:** WebMCP Storefront, WebMCP Calendar, WebMCP Slides, Bakery Assistant
- **Kurulum:** `$ npm install @runtypelabs/persona`

### Gerçek Kullanım Senaryoları
- E-ticaret: AI agent'ın ürün sayfasında işlem yapması
- Randevu sistemleri: Agent'ın takvim uygulamasında rezervasyon oluşturması

## 3. ArgusRed/argus — AI Pen Testing Modeli

### Ne Yapıyor?
"Red team için eğitilmiş model — reddetmek yerine pentest yapıyor." 90 HN puanı, Show HN'de viral oldu. Geleneksel LLM'ler "bu zararlı olur" deyip reddeder. Argus tersine çalışıyor: güvenlik açığı buluyor, exploit yazıyor,patch öneriyor.

### Herkesin Kaçırdığı Nokta
**"Refusal yerine action" = yeni model kategorisi.** Geleneksel RLHF modelleri zararlı istekleri reddeder. Argus tam tersi — güvenlik profesyonellerinin "benim işim güvenlik testi, bu istekler zararlı değil" demesine gerek yok, model bunu ayırt edebiliyor. Bu, "alignment tax" (güvenlik nedeniyle performans kaybı) sorununu çözüyor.

### Herkesin Kaçırdığı Nokta #2
**Kurumsal güvenlik ekipleri için kritik.** Bugün red team'ler manuel penetration test yapıyor. Argus ile otomatik tarama + insan onayı = 10x hız. Ancak: "otomatik exploit üretimi" = yeni güvenlik riskleri. Bu denge önümüzdeki dönem büyük tartışma yaratacak.

### Teknik Detaylar
- **Puan:** 90 HN pts (2026-06-20)
- **Yaklaşım:** Post-training ile "refusal → action" dönüşümü
- **Kullanım:** CLI tabanlı, güvenlik tarayıcısı

### Gerçek Kullanım Senaryoları
- DevSecOps: CI/CD pipeline'ında otomatik güvenlik taraması
- Bug bounty hunter'lar: Hızlı başlangıç taraması
- Kurumsal red team: Sürekli güvenlik değerlendirmesi

**Kaynak:** https://www.argusred.com/cli

## 4. airgap (sauleau) — Secrets Isolation for AI Agents

### Ne Yapıyor?
Linux namespace'leri içinde çalışan bir wrapper. AI agent'lar .env dosyalarındaki gerçek değerleri görmüyor — sadece "secret var" sinyalini alıyor. npm install bile gated — hangi dosyalara erişeceğini agent değil, kullanıcı kontrol ediyor.

### Herkesin Kaçırdığı Nokta
**"AI agent install npm" = yeni supply chain saldırı vektörü.** Miasma worm tam da bunu istismar etti. airgap'in yaklaşımı: agent "bu paketi kur" dediğinde, wrapper "bu paket hangi dosyalara yazacak?" diye soruyor ve kullanıcı onay verene kadar bekliyor. Agent root yetkisi olsa bile, namespace隔离 sayesinde gerçek dosyalara erişemiyor.

### Herkesin Kaçırdığı Nokta #2
**Linux only = henüz erken ama yön doğru.** macOS desteği "work in progress." Ama konsept test edildi — bu yaklaşım gelecekte tüm platformlarda standart olabilir. Benzer projeler: Cordium (GitHub'da 2 HN puanı), Guardian Runtime.

### Teknik Detaylar
- **Platform:** Linux (macOS yakında)
- **Teknoloji:** Linux namespaces, file hiding
- **Kullanım:** `airgap run <command>` — komutu isolated namespace'de çalıştırır

### Gerçek Kullanım Senaryoları
- Geliştirici makinelerinde agent güvenliği
- CI/CD pipeline'larında minimiz edilmiş agent yetkileri
- Production ortamlarında agent erişim kontrolü

**Kaynak:** https://sauleau.com/notes/airgap-security-for-the-modern-ai-age.html

## 5. thu-nmrc/openloop — Universal Loop Engineering Framework

### Ne Yapıyor?
Tsinghua Üniversitesi'nden açık kaynak proje. AI agent döngülerini (play-test-fix-verify-improve) yönetmeye yarıyor. Logs, heartbeats, baselines, guardrails ve auditable stop conditions ile agent'ların ne zaman duracağını, nasıl test edileceğini tanımlayabiliyorsunuz.

### Herkesin Kaçırdığı Nokta
**"Loop engineering" = yeni disiplin.** Bugün agent framework'ler "nasıl çalıştırırım"a odaklanıyor. OpenLoop "nasıl test ederim, nasıl durdururum, nasıl doğrularım"a odaklanıyor. Bu, agent'ların production'da kontrolsüz çalışmasını engellemek için kritik.

### Herkesin Kaçırdığı Nokta #2
**"Auditable stop conditions" = enterprise gereksinimi.** Finans, sağlık, hukuk sektörlerinde regulator'lar agent'ların nasıl karar verdiğini, ne zaman durduğunu görmek istiyor. OpenLoop'un stop conditions'ı = tam audit trail.

### Teknik Detaylar
- **Dil:** Python
- **Oluşturulma:** 2026-06-10
- **Topics:** AI agent loop management, testing, verification
- **Geliştirici:** Tsinghua University NMRC Lab

### Gerçek Kullanım Senaryoları
- Enterprise agent deployment: Regulator uyumluluğu
- Mission-critical automation: Hata durumunda nasıl durur?
- Test-driven agent development: Agent'ı deploy etmeden önce doğrula

**Kaynak:** https://github.com/thu-nmrc/openloop

## 6. yaodub/cast — Multi-User Claude Agent Harness

"Kendi makinende agent takımın." TypeScript ile yazılmış, multi-user Claude agent orchestration aracı. Birden fazla agent'ı aynı makinede, container-based izolasyonla çalıştırıyor.

### Herkesin Kaçırdığı Nokta
**"Multi-user" = tek başına developer aracı DEĞİL.** Ekipler için tasarlanmış — birden fazla developer aynı agent takımını kullanabilir, her birinin container'ı izole. Bu, "her developer kendi Claude Code instance'ını açıyor" yaklaşımından farklı: merkezi yönetim, izleme, maliyet kontrolü.

### Herkesin Kaçırdığı Nokta #2
**MCP desteği = genişletilebilirlik.** cast'ın MCP entegrasyonu = farklı agent'lar farklı tool'lara erişebilir. Bir agent sadece code review yaparken, diğeri deployment yapabilir — ve ikisi aynı projede çalışabilir.

## Herkesin Kaçırdığı Nokta — Özet

### #1: "API Key Güvenliği" = Yeni Standart Olacak
Cloak, Airgap, Cordium — üç farklı yaklaşım, aynı problem: agent'ların secrets'lere erişimi. 2026'nın ikinci yarısında her agent framework'ü "built-in secrets management" sunacak. Bugün bu bir "extra" — yarın zorunlu olacak.

### #2: "Agent UI" = Kullanıcı Tarafı Kaçırılıyor
Persona.js, Cast, WebMCP — tümü geliştirici tarafına odaklanıyor. Ama asıl değer: kullanıcının web sitesinde agent'ın çalışması. E-ticaret, randevu, rezervasyon — "benim adıma hareket eden agent" = kullanıcının gördüğü ilk AI Native uygulama olacak.

### #3: "Agent Pen Testing" = Yeni Savaş Alanı
Argus gibi modeller "refusal → action" dönüşümü yapıyor. Bu hem güvenlik profesyonelleri için güçlü hem de kötüye kullanım riski yüksek. 2026'da "AI agent güvenlik" = en hızlı büyüyen alt sektör.
